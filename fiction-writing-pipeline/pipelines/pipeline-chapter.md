# Pipeline Chapter — 단일 챕터 작성

## 개요
- **대상**: 기존 작품의 단일 챕터 또는 독립 씬 묶음
- **컨펌 횟수**: 2회
- **참여 에이전트**: fw-writer, fw-editor
- **주요 산출물**: 챕터 초고 + 편집 완성본 + DOCX
- **전제 조건**: 기존 캐릭터 시트/씬 카드 존재 또는 작가가 직접 제공

---

## 실행 순서

### 📥 Step 0: 시작
```
작가 입력: "Chapter — [설정 정보] + [이번 챕터 목표]"
```

작가가 제공해야 할 정보:
- 주인공 이름 + 핵심 설정 (1-3문장)
- 이 챕터에서 일어나야 할 일 (목표)
- 톤/분위기
- 이전 챕터 요약 (있을 경우)

→ `feedback.md` 로드
→ Chapter 파이프라인 시작

---

### 📋 Step 1: 씬 계획 + 집필
```
호출: fw-writer
  - 씬 계획 (3-8씬) 수립 후 바로 집필
  - skills/draft-writer.md
출력: output/drafts/chapter-[N]-[제목]-[YYYYMMDD].md
```

챕터 단위 특화:
- 씬 카드 전체 작성 대신 간략 씬 계획 (씬당 2-3줄)
- 계획 직후 바로 집필 진행
- 3,000-5,000자 목표

---

### ⭐ 컨펌 1 — 초고 확인

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
✍️ 챕터 초고 완성 — 작가 확인 필요
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[초고 전문 또는 앞부분 미리보기]
분량: 약 [X]자

✅ 승인하면 편집 단계로 넘어갑니다
✏️ 방향 수정이 필요하면 알려주세요
```

---

### 🔧 Step 2: 편집 (집중 버전)
```
호출: fw-editor
  - skills/prose-editor.md (문장 편집 중심)
  - 필요 시 skills/character-editor.md (설정과 충돌 점검)
입력: 초고
출력: output/final/chapter-[N]-[제목]-final.md
```

챕터 편집 특화:
- 구조 편집 생략 (챕터 단위라 구조 변경 없음)
- 캐릭터 편집: 기존 설정과 충돌 여부만 점검
- 문장 편집: 챕터 전체 집중

---

### ⭐ 컨펌 2 — 최종 승인

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
✍️ 편집 완료 — 최종 확인
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[편집 전후 주요 변경 요약]

✅ 승인하면 DOCX로 저장합니다
✏️ 추가 수정이 필요하면 알려주세요
```

---

### 📦 Step 3: DOCX 저장 + 마무리
```
호출: skills/save-to-docx.md
출력: output/final/chapter-[N]-[제목]-final-[YYYYMMDD].docx
```

```
챕터 완성 🎉
output/final/chapter-[N]-[제목]-final-[YYYYMMDD].docx
```

---

## Chapter 파이프라인 활용 팁

- 장편소설 작업 중 특정 챕터만 다시 쓸 때 사용
- 기존 캐릭터 시트가 있으면 `output/characters/`에 저장 후 참조
- 씬 카드가 있으면 `output/structure/`에 저장 후 참조

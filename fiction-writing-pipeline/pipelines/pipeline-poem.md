# Pipeline Poem — 시 쓰기

## 개요
- **대상**: 단편 시, 연작시, 시조, 자유시 등 모든 시 형식
- **컨펌 횟수**: 2회
- **참여 에이전트**: fw-poet
- **주요 산출물**: 시상 문서 + 시 초고 + 퇴고 완성본 + DOCX
- **특이점**: 가장 짧은 파이프라인 — 시상 → 초고 → 퇴고 → 저장

---

## 실행 순서

### 📥 Step 0: 시작
```
작가 입력: "Poem — [시상/감정/단어/장면]으로 시 써줘"
```
→ `feedback.md` 로드
→ 형식 미지정 시 자유시로 진행 (이후 변경 가능)

**입력 예시**:
- "Poem — 첫눈 내리던 날 헤어진 사람"
- "Poem — 빈 그릇, 자유시로"
- "Poem — 어머니를 생각하며, 시조로"
- "Poem — 퇴근길 지하철, 현대시"

---

### 💡 Step 1: 시상 구체화 + 초고 집필
```
호출: fw-poet
  → 시상 구체화 (핵심 이미지 · 감정 온도 · 형식 확정)
  → skills/poem-writer.md → 초고 집필
출력:
  - output/concepts/poem-concept-[제목]-[YYYYMMDD].md
  - output/drafts/poem-[제목]-[YYYYMMDD].md
```

---

### ⭐ 컨펌 1 — 초고 확인

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
✍️ 시 초고 완성 — 작가 확인 필요  [컨펌 1/2]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[시 전문]

형식: [자유시/시조 등] | 행수: [N]행
핵심 이미지: [한 줄 설명]

✅ 승인하면 → 퇴고를 시작합니다
✏️ 수정하면 → 바꿀 방향을 알려주세요
   (예: "3연을 더 강하게", "마지막 행 다시", "형식을 시조로")
```

작가 응답 대기 → 승인 또는 수정 반영 후 Step 2 진행

---

### 🔧 Step 2: 퇴고
```
호출: fw-poet → skills/poem-editor.md
입력: output/drafts/poem-[제목].md
출력:
  - output/drafts/poem-edit-notes-[제목]-[YYYYMMDD].md
  - output/final/poem-[제목]-final.md
```

퇴고 집중 항목:
- 이미지 강화 (클리셰 제거)
- 밀도 — 불필요한 행·단어 삭제
- 마지막 행의 여운
- 리듬 흐름 (소리 내어 읽기 기준)

---

### ⭐ 컨펌 2 — 최종 승인

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
✍️ 퇴고 완료 — 최종 확인  [컨펌 2/2]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[퇴고 완성본 전문]

주요 수정: [퇴고 메모 요약 2-3줄]

✅ 승인하면 → DOCX로 저장합니다
✏️ 추가 수정이 필요하면 알려주세요
```

---

### 📦 Step 3: DOCX 저장 + 마무리
```
호출: skills/save-to-docx.md
출력: output/final/poem-[제목]-final-[YYYYMMDD].docx
```

```
시 완성 🎉
output/final/poem-[제목]-final-[YYYYMMDD].docx
```

---

## 연작시 옵션

여러 편을 묶는 연작시를 원하면:
```
"Poem — [주제]로 연작시 [N]편 써줘"
```
→ 각 편마다 컨펌 1 진행 → 전체 묶음으로 최종 컨펌 → DOCX 저장

---

## 지원 형식 요약

| 형식 | 입력 키워드 |
|------|-----------|
| 자유시 | "자유시로", 형식 미지정 |
| 시조 | "시조로" |
| 소네트 | "소네트로" |
| 산문시 | "산문시로" |
| 하이쿠 | "하이쿠로" |
| 연작시 | "[N]편 연작으로" |

# structural-editor — 구조 편집 스킬 (1차 편집)

## 역할
초고 전체를 읽고 이야기 구조의 문제를 찾아 수정한다.
씬 순서, 페이스, 3막 균형, 빠진/중복 씬을 처리하는 것이 목표.

## 입력
- `output/drafts/` 모든 챕터 파일
- `output/structure/scene-cards-[제목].md` (원래 계획과 비교)
- `feedback.md`

## 출력
- `output/drafts/structural-notes-[제목]-[YYYYMMDD].md` (편집 메모)
- 수정된 초고 (각 챕터 파일 업데이트)

---

## 구조 편집 체크리스트

### 전체 구조
- [ ] 1막이 전체의 25% 이내인가?
- [ ] 2막 중간점(Midpoint)이 명확한가?
- [ ] 최저점(All Is Lost)이 클라이맥스 직전에 배치되어 있는가?
- [ ] 3막 해소가 지나치게 길지 않은가?

### 씬별 점검
각 씬에 대해:
- [ ] 이 씬을 삭제해도 이야기가 진행되는가? → 삭제 또는 통합 검토
- [ ] 씬의 시작과 끝에 변화가 있는가?
- [ ] 연속 3씬 이상 같은 텐션 레벨인가? → 완급 조절 필요

### 페이스
- [ ] 이야기가 너무 빠르게 진행되는 구간 (설명 보강 필요)
- [ ] 이야기가 늘어지는 구간 (씬 압축 또는 삭제)

### 플롯 홀
- [ ] 인과관계가 끊기는 지점
- [ ] 설명 없이 등장하는 요소
- [ ] 복선이 회수되지 않는 요소

---

## 편집 메모 형식

```markdown
# 구조 편집 메모: [제목]
날짜: YYYY-MM-DD

## 전체 평가
[이야기 구조의 강점과 주요 문제 2-3문장]

## 수정 사항

### 삭제/통합 씬
- 씬 [N]: [이유]

### 추가 필요 씬
- [위치]: [이유]

### 순서 변경
- 씬 [A]와 씬 [B] 교환: [이유]

### 페이스 조절
- 챕터 [N]: [늘리기/줄이기] — [이유]

## 다음 단계 (캐릭터 편집) 주의사항
- [구조 편집 후 캐릭터 편집자가 알아야 할 사항]
```

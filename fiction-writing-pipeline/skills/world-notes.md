# world-notes — 세계관 노트 생성 스킬

## 역할
확정된 로그라인을 바탕으로 이야기의 무대가 되는 세계관을 체계적으로 구축한다.
설정 과잉 없이, 이야기에 필요한 세계관만 만드는 것이 원칙.

## 입력
- `output/concepts/logline-[제목].md`
- `feedback.md` (세계관 스타일 참조)
- 작가의 추가 설정 지시 (있을 경우)

## 출력
`output/worldbuilding/worldnotes-[제목]-[YYYYMMDD].md`

---

## 세계관 노트 작성 구조

```markdown
# 세계관 노트: [제목]
날짜: YYYY-MM-DD | 장르: [장르]

---

## 1. 세계 한 줄 정의
[이 세계를 가장 잘 설명하는 한 문장]

## 2. 시공간 배경
- **시대**: [실제 역사 / 대안 역사 / 근미래 / 판타지 등]
- **장소**: [국가, 도시, 지역 — 실제 또는 창작]
- **기술 수준**: [마법 / 현대 / SF 등]

## 3. 핵심 규칙 (3-5개)
이 세계가 현실과 다른 가장 중요한 규칙들.

- **규칙 1**: [명칭] — [설명. 이 규칙이 이야기에 주는 의미]
- **규칙 2**:
- **규칙 3**:

## 4. 사회 구조
- **권력 체계**: [지배 계급, 정치 구조]
- **경제/계급**: [빈부 격차, 계층 이동 가능성]
- **핵심 갈등 구조**: [사회적 모순이나 대립축]

## 5. 주요 장소
각 장소의 기능과 분위기:

| 장소명 | 기능 | 분위기 | 등장 챕터 |
|-------|------|--------|---------|
| [장소 1] | | | |
| [장소 2] | | | |

## 6. 문화 & 관습
- [이 세계 특유의 문화 요소 — 독자가 낯설게 느낄 것]

## 7. 미확정 요소 (TODO)
- [ ] [아직 결정하지 않은 설정]
```

---

## 세계관 구축 원칙

1. **설정은 이야기를 위해 존재한다** — 아무리 멋진 설정도 이야기에 필요 없으면 삭제
2. **냉장고 법칙** — 독자가 냉장고 문을 열기 전까지 냉장고 내부를 설명하지 않아도 된다
3. **내적 일관성** — 규칙을 정했으면 예외를 만들지 않는다 (예외는 이야기의 핵심일 때만)
4. **독자의 경험** — 설정 덩어리 대신 장면 속에서 자연스럽게 드러낸다

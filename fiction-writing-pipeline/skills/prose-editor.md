# prose-editor — 문장 편집 스킬 (3차 편집)

## 역할
구조와 캐릭터 편집이 완료된 원고의 문장 수준을 높인다.
리듬, 어휘 선택, 클리셰 제거, 맞춤법을 다듬는 최종 편집 단계.

## 입력
- 캐릭터 편집 완료된 원고 (전체)
- `output/drafts/structural-notes-[제목].md`
- `output/drafts/character-notes-[제목].md`
- `feedback.md` (문체 취향 최우선 반영)

## 출력
- `output/drafts/prose-notes-[제목]-[YYYYMMDD].md` (편집 메모)
- `output/final/[제목]-final.md` (최종 편집 완료 통합본)

---

## 문장 편집 체크리스트

### 리듬과 길이
- [ ] 같은 길이의 문장이 3개 이상 연속되지 않는가?
- [ ] 긴 문장과 짧은 문장이 적절히 섞이는가?
- [ ] 긴박한 장면에서 문장이 짧아지는가?

### 어휘
- [ ] 한 단락에서 같은 단어가 2회 이상 반복되지 않는가?
- [ ] 클리셰 표현 제거 (눈물이 주르륵, 심장이 쿵, 눈이 번쩍 등)
- [ ] 과도한 부사/형용사 제거

### 쇼 돈 텔
- [ ] "~하게 느껴졌다", "~인 것 같았다" → 구체적 행동/감각으로
- [ ] 감정 직접 서술 → 행동/표정/신체 반응으로

### 문법 & 맞춤법
- [ ] 시제 일관성
- [ ] 맞춤법·띄어쓰기
- [ ] 문장 호응 오류

### 대화
- [ ] 대화 태그 과잉 ("격렬하게 외쳤다" → "말했다")
- [ ] 대화와 행동 비트의 균형

## 편집 메모 형식

```markdown
# 문장 편집 메모: [제목]
날짜: YYYY-MM-DD

## 전체 문체 평가
[이 원고의 문장 수준 강점과 개선점]

## 주요 수정 패턴
- 클리셰: [발견된 클리셰와 수정 방향]
- 반복 어휘: [반복 어휘 목록]
- 리듬 문제 구간: 챕터 [N]

## 작가 문체 보존 결정 사항
- [의도적 선택으로 판단하여 수정하지 않은 부분]
```

---

## 문장 편집 원칙

1. **작가의 목소리를 지우지 않는다** — 균형 잡힌 문장보다 고유한 문장이 낫다
2. **이유 없는 수정 금지** — 더 낫다는 확신이 없으면 손대지 않는다
3. **소리 내어 읽기** — 어색한 리듬은 읽으면 바로 드러난다

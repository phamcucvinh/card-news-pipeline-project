# 미니멀 일기장 데스크톱 애플리케이션 설계 문서

## 📋 프로젝트 개요

### 목적
사용자가 일상의 생각과 경험을 기록할 수 있는 미니멀한 데스크톱 일기장 애플리케이션

### 핵심 가치
- **단순함**: 복잡한 기능 없이 글쓰기에만 집중
- **프라이버시**: 로컬 저장, 외부 서버 불필요
- **집중력**: 방해 요소 없는 깨끗한 인터페이스
- **접근성**: 직관적이고 배우기 쉬운 UI

### 타겟 사용자
- 일기 작성 습관을 만들고 싶은 사람
- 복잡한 기능보다 단순함을 선호하는 사람
- 데이터 프라이버시를 중요하게 생각하는 사람

## 🏗️ 시스템 아키텍처

### 전체 구조
```
┌─────────────────────────────────────────┐
│         Presentation Layer              │
│  ┌────────────┐      ┌──────────────┐  │
│  │   UI       │◄────►│  View Logic  │  │
│  │ Components │      │   (State)    │  │
│  └────────────┘      └──────────────┘  │
└──────────────────┬──────────────────────┘
                   │
┌──────────────────▼──────────────────────┐
│         Application Layer               │
│  ┌────────────┐      ┌──────────────┐  │
│  │  Diary     │      │   Settings   │  │
│  │  Manager   │      │   Manager    │  │
│  └────────────┘      └──────────────┘  │
└──────────────────┬──────────────────────┘
                   │
┌──────────────────▼──────────────────────┐
│          Data Layer                     │
│  ┌────────────┐      ┌──────────────┐  │
│  │   Local    │      │    File      │  │
│  │  Storage   │      │   System     │  │
│  └────────────┘      └──────────────┘  │
└─────────────────────────────────────────┘
```

### 레이어 설명

#### 1. Presentation Layer (프레젠테이션 레이어)
- **책임**: UI 렌더링 및 사용자 상호작용 처리
- **주요 컴포넌트**:
  - `EditorView`: 일기 작성/편집 화면
  - `CalendarView`: 날짜별 일기 목록
  - `SidebarView`: 네비게이션 사이드바
  - `SettingsView`: 앱 설정 화면

#### 2. Application Layer (애플리케이션 레이어)
- **책임**: 비즈니스 로직 처리
- **주요 모듈**:
  - `DiaryManager`: 일기 CRUD 작업 관리
  - `SearchManager`: 일기 검색 기능
  - `SettingsManager`: 앱 설정 관리

#### 3. Data Layer (데이터 레이어)
- **책임**: 데이터 영속성 관리
- **주요 모듈**:
  - `LocalStorage`: 일기 데이터 저장/불러오기
  - `FileSystem`: 파일 시스템 접근

## 💾 데이터 구조

### 데이터 모델

#### DiaryEntry (일기 항목)
```typescript
interface DiaryEntry {
  id: string;              // UUID v4
  date: string;            // ISO 8601 형식 (YYYY-MM-DD)
  createdAt: number;       // Unix timestamp (ms)
  updatedAt: number;       // Unix timestamp (ms)
  content: string;         // 본문 (Markdown)
  wordCount: number;       // 단어 수
}
```

#### Settings (설정)
```typescript
interface Settings {
  version: string;         // 설정 버전
  appearance: {
    fontSize: number;      // 14-24px
    lineHeight: number;    // 1.4-2.0
    fontFamily: string;    // 폰트 이름
  };
  editor: {
    autoSave: boolean;     // 자동 저장
    saveInterval: number;  // 저장 간격 (ms)
    spellCheck: boolean;   // 맞춤법 검사
  };
  privacy: {
    confirmDelete: boolean; // 삭제 확인
  };
}
```

## 🛠️ 기술 스택

### Frontend Framework
**선택: Electron + React + TypeScript**

**이유:**
- ✅ 크로스 플랫폼 지원 (Windows, macOS, Linux)
- ✅ 웹 기술 활용 (빠른 개발)
- ✅ 풍부한 생태계
- ✅ 네이티브 파일 시스템 접근
- ✅ TypeScript로 타입 안정성 확보

### UI Library
**선택: React + CSS Modules**

**이유:**
- ✅ 컴포넌트 기반 구조
- ✅ 가볍고 유연함
- ✅ CSS Modules로 스타일 격리
- ❌ 불필요한 UI 라이브러리 제외 (미니멀 철학)

### 상태 관리
**선택: React Context API + useReducer**

**이유:**
- ✅ 간단한 앱에 적합
- ✅ 추가 라이브러리 불필요
- ✅ 충분한 상태 관리 기능

### 데이터 저장
**선택: 파일 시스템 (JSON)**

**이유:**
- ✅ 데이터베이스 불필요 (단순함)
- ✅ 백업 및 이동 용이
- ✅ 사용자가 직접 접근 가능
- ✅ Git 등으로 버전 관리 가능

### 추가 라이브러리
```json
{
  "date-fns": "날짜 처리",
  "uuid": "고유 ID 생성",
  "electron-store": "설정 저장"
}
```

## 🔐 보안 및 프라이버시

### 데이터 보안
- **로컬 저장**: 모든 데이터는 사용자 컴퓨터에만 저장
- **암호화 없음**: 1차 버전에서는 평문 저장 (추후 선택적 추가)
- **네트워크 통신 없음**: 인터넷 연결 불필요

### 프라이버시 정책
- 사용자 데이터 수집 안 함
- 분석 도구 사용 안 함
- 외부 서버 통신 안 함

# tone-transformer — 문체 변환 스킬

## 역할
완성된 텍스트를 채널·목적·독자에 맞는 문체로 변환한다.
같은 내용을 격식체·구어체·친근체·전문가체 등으로 재작성하는 것이 목표.

## 입력
- 원본 텍스트
- 변환 목표 문체 (아래 유형 참조)
- 채널 및 독자 (선택)
- `feedback.md`

## 출력
채팅으로 직접 제시

---

## 문체 유형

### 1. 구어체 (일상·친근)
- **특징**: 말하듯 씀, 짧은 문장, 반말 느낌 (하지만 존댓말)
- **채널**: 인스타그램, 스레드, 틱톡
- **예시**: "사실 저도 처음엔 몰랐어요. 그냥 엑셀만 쓰면 되는 줄 알았거든요."

### 2. 격식체 (전문·신뢰)
- **특징**: 완결된 문장, 논리적 구조, 데이터 중심
- **채널**: 링크드인, 브런치, 보고서
- **예시**: "마케팅 자동화 도입 시 반복 업무 시간을 평균 62% 절감할 수 있습니다."

### 3. 교육체 (설명·가이드)
- **특징**: 단계별 설명, "~하세요" 형식, 예시 풍부
- **채널**: 뉴스레터, 유튜브 스크립트, 블로그
- **예시**: "먼저 파이썬을 설치하세요. 그 다음 이렇게 입력하면 됩니다."

### 4. 스토리텔링체 (감성·공감)
- **특징**: 경험담 중심, 감정 묘사, 기승전결
- **채널**: 뉴스레터, 브런치, 커리어 콘텐츠
- **예시**: "3년 전, 저는 매일 밤 같은 엑셀 작업을 반복하고 있었습니다."

### 5. 카피라이팅체 (임팩트·간결)
- **특징**: 짧고 강렬한 문장, 숫자/대비 구조
- **채널**: 광고, 카드뉴스 표지, 썸네일
- **예시**: "3시간짜리 업무, 3분으로 줄이는 법"

---

## 변환 요청 형식
```
원본: [텍스트]
변환: [목표 문체]
채널: [채널명] (선택)
```

---

## 변환 원칙
1. **핵심 정보 유지**: 사실·수치 변경 금지
2. **길이 조정 허용**: 문체에 맞게 길이 변환 가능
3. **JJ 보이스**: 어떤 문체든 JJ 목소리 유지 (`feedback.md` 참조)
4. **2개 제시**: 변환본 + 간략한 변환 포인트 설명

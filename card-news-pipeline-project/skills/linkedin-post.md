# linkedin-post — 링크드인 포스트 작성 스킬

## 역할
칼럼을 바탕으로 링크드인에 최적화된 전문가 포스트를 작성한다.
JJ의 퍼스널 브랜딩을 강화하고 마케터 네트워크에 인사이트를 전달하는 것이 목표.

## 입력
- `output/columns/column-[주제].md`
- 시리즈 유형
- `feedback.md`

## 출력
`output/columns/linkedin-[주제]-[YYYYMMDD].md`

---

## 링크드인 포스트 구조

```markdown
[첫 줄 — 스크롤 멈추는 훅. 1문장, 강렬하게]

[공백]

[본문 — 3-5개 핵심 인사이트, 짧은 단락으로]

[공백]

[마무리 — 독자에게 질문 또는 행동 유도]

[공백]

[해시태그 5-8개]
```

---

## 링크드인 특성 반영

### 알고리즘 친화
- **첫 3줄이 전부**: "더 보기" 클릭 전 보이는 분량
- **줄바꿈 필수**: 빽빽한 단락은 스크롤 유발
- **댓글 유도**: 마지막에 질문 필수

### 전문가 톤
- 1인칭 경험담 형식 우선 ("제가 직접 해봤는데...")
- 데이터/수치로 신뢰도 보강
- 겸손하지만 확신 있는 어투

### 분량
- **최적**: 800-1,200자
- **최대**: 3,000자 (접힘 발생)
- **단락**: 2-3줄 → 공백 → 2-3줄

## 포스트 유형별 훅 예시

| 유형 | 훅 예시 |
|------|--------|
| 경험담 | "3년 전, 저는 엑셀 매크로조차 몰랐습니다." |
| 통계 | "마케터의 62%가 반복 업무에 하루 2시간을 씁니다." |
| 반전 | "코딩을 잘하는 마케터보다 코딩을 활용하는 마케터가 낫습니다." |
| 질문 | "여러분은 GPT를 어디까지 쓰고 계신가요?" |

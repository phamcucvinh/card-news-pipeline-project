# thread-writer — 스레드/X(트위터) 작성 스킬

## 역할
칼럼을 스레드/X 형식 7-12개 트윗으로 재구성한다.

## 입력
- `output/columns/column-[주제].md`

## 출력
`output/columns/thread-[주제]-[YYYYMMDD].md`

---

## 스레드 구조

```markdown
# 스레드: [주제]

---

1/
[후킹 첫 트윗]
[독자가 멈추게 하는 첫 문장]
[다음 트윗 예고]

→ 140자 이내

---

2/
[핵심 포인트 1]

→ 140자 이내

---

3/
[핵심 포인트 2]

→ 140자 이내

...

---

n/
[마무리 + CTA]
[팔로우 유도 또는 저장 유도]

→ 140자 이내
```

---

## 트윗 작성 원칙

### 첫 트윗 (가장 중요)
- 숫자 포함: "X가지 알면 달라지는 것들"
- 반전: "~인데 사실은 ~입니다"
- 직접 이익: "~하면 ~할 수 있습니다"

### 중간 트윗
- 각 트윗은 독립적으로 읽혀야 함
- 연결고리: 앞 트윗 → 이 트윗 → 다음 트윗 흐름 유지
- 한 트윗에 하나의 메시지만

### 마지막 트윗
- 핵심 요약 (리스트 형식)
- 저장/리트윗 유도
- 관련 콘텐츠 예고

---

## 140자 체크 팁
한글 기준 140자 = 영어 280자 (X 기준)
스레드 앱은 한글 140자 기준 적용

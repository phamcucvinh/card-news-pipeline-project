# repurpose-writer — 콘텐츠 재활용 스킬

## 역할
기존 완성 콘텐츠를 다른 형식 또는 채널에 맞게 재가공한다.
한 번 만든 콘텐츠의 수명을 늘리고 채널별 최적화를 통해 도달을 극대화하는 것이 목표.

## 입력
- 원본 콘텐츠 파일 경로
- 변환 목표 형식 (아래 매트릭스 참조)
- `feedback.md`

## 출력
`output/columns/repurpose-[원본주제]-[형식]-[YYYYMMDD].md`

---

## 재활용 매트릭스

| 원본 → | 카드뉴스 | 스레드 | 링크드인 | 쇼츠 스크립트 | 뉴스레터 |
|--------|---------|--------|---------|------------|---------|
| 칼럼 | ✅ | ✅ | ✅ | ✅ | ✅ |
| 카드뉴스 | — | ✅ | ✅ | ✅ | ✅ |
| 뉴스레터 | ✅ | ✅ | ✅ | ✅ | — |
| 스레드 | ✅ | — | ✅ | ✅ | ✅ |

---

## 재활용 전략

### 칼럼 → 카드뉴스
- 소제목 3개 = 슬라이드 3장
- 핵심 수치 강조 슬라이드 추가
- `quote-extractor.md` 연동으로 표지 문구 추출

### 칼럼 → 스레드
- 소제목 1개 = 스레드 1개
- 데이터/사례는 독립 스레드로
- 마지막 스레드: CTA + 전체 링크

### 칼럼 → 쇼츠
- 칼럼의 핵심 3포인트 = 쇼츠 본문
- `youtube-shorts-script.md` 연동

### 오래된 콘텐츠 업데이트 재활용
- 6개월 이상 된 콘텐츠 → 최신 데이터로 업데이트
- "업데이트 버전" 명시
- 원본과 다른 각도로 접근

---

## 재활용 시 주의사항
1. **중복 느낌 금지**: 같은 콘텐츠라도 채널별로 다르게 느껴져야 함
2. **업데이트 확인**: 수치·사례가 최신인지 확인
3. **원본 출처 유지**: 내부 링크 또는 "이전 글에서" 언급
4. **간격**: 같은 콘텐츠 재활용은 최소 4주 후

---

## 재활용 우선순위
```
가장 반응 좋았던 콘텐츠 → 즉시 재활용
시즌 관련 콘텐츠 → 시즌마다 재활용
에버그린 콘텐츠 → 분기별 재활용
```

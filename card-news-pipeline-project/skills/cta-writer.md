# cta-writer — CTA 문구 생성 스킬

## 역할
콘텐츠 목적과 채널에 맞는 클릭 유도(CTA) 문구 3안을 작성한다.
독자가 자연스럽게 다음 행동을 하도록 유도하는 것이 목표.

## 입력
- 콘텐츠 주제 및 시리즈
- CTA 목표 (팔로우 / 뉴스레터 구독 / 링크 클릭 / 댓글 / 저장)
- 채널 (인스타그램 / 링크드인 / 스레드 / 이메일)
- `feedback.md`

## 출력
채팅으로 직접 제시

---

## CTA 유형별 3안 구성

### 팔로우 유도
```
A (가치 제안형): "매주 마케터를 위한 실전 팁을 올립니다. 팔로우하면 놓치지 않아요 👉"
B (커뮤니티형): "같이 성장하는 마케터들이 모여 있어요. 함께해요!"
C (호기심형): "다음 주제는 더 실용적이에요. 팔로우하고 기다려보세요 😊"
```

### 뉴스레터 구독 유도
```
A (혜택형): "매주 월요일, 이런 인사이트를 이메일로 받아보세요. 구독 링크 바이오에 있어요"
B (수치형): "3,000명의 마케터가 구독 중인 뉴스레터, 무료예요"
C (시험형): "한 달만 구독해보세요. 언제든 해지 가능해요"
```

### 댓글 유도
```
A (질문형): "여러분은 어떻게 생각하세요? 댓글로 알려주세요 👇"
B (경험 공유형): "비슷한 경험 있으신가요? 댓글에서 얘기 나눠요"
C (투표형): "A vs B, 어느 쪽이 더 효과적일까요?"
```

### 저장 유도
```
A (실용형): "나중에 써먹을 것 같으면 저장해두세요 🔖"
B (공유형): "도움이 됐다면 저장하고 필요한 분께 공유해주세요"
C (참고형): "업무에 바로 쓸 수 있는 내용이에요. 저장 추천!"
```

## 채널별 CTA 특성

| 채널 | 특성 | 피해야 할 것 |
|------|------|------------|
| 인스타그램 | "바이오 링크" 활용 | "링크 클릭" (직접 링크 불가) |
| 링크드인 | 댓글 알고리즘 활용 | 외부 링크 본문 삽입 |
| 스레드 | 마지막 스레드에 배치 | 과도한 CTA |
| 이메일 | 버튼 형식 | 링크 2개 이상 (분산) |

---

## 좋은 CTA 원칙
1. **한 개만**: 하나의 콘텐츠에 CTA는 1개
2. **자연스럽게**: 강요하지 않고 가치를 먼저
3. **구체적으로**: "좋아요 눌러주세요"보다 "저장해두세요"
4. **이유 제시**: 왜 해야 하는지 한 줄 설명

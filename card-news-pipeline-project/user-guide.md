# 카드뉴스 파이프라인 사용자 가이드

> AI 에이전트 기반 카드뉴스 자동화 시스템

---

## 1. 프로젝트 소개

### 이 프로젝트는 무엇인가요?

카드뉴스 파이프라인은 콘텐츠 아이디어만 제시하면, AI 에이전트들이 리서치부터 설계, 제작, 검증까지 전 과정을 자동화하는 시스템입니다.

**할 수 있는 것**:
- 웹 기반 리서치 및 팩트 수집
- 칼럼/콘텐츠 자동 작성
- 비주얼 카드뉴스 HTML 슬라이드 생성
- PNG 이미지 추출
- SNS 캡션 자동 작성
- QA 검증 및 품질 관리

### 4가지 시리즈

| 시리즈 | 이름 | 특징 | 산출물 |
|--------|------|------|--------|
| **A** | 코딩하는 마케터 | 주제 기반 칼럼 | 카드뉴스 + 롱폼 + 스레드 |
| **B** | 알쓸IT잡 | IT 용어 설명 | 카드뉴스 비주얼 + MP4 표지 |
| **C** | 커리어 레터 | 원문 변환 | 카드뉴스 시네마틱 |
| **D** | 코마레터 | 뉴스레터 | 뉴스레터 + 멀티채널 배포본 |

### 핵심 설계 원칙

1. **Human-in-the-Loop**: 핵심 결정 포인트마다 사용자 확인
2. **역할 분리**: 각 스킬이 하나의 전문 역할에 집중
3. **누적 학습**: `feedback.md`에 사용자 취향 저장 후 반영

## 2. 시작하기

### 사전 요구사항

- Claude Code (AI 에이전트 인터페이스)
- 인터넷 연결 (웹 리서치 필수)
- Python 3.7+ (PNG 변환 스크립트 실행 시)

### 폴더 구조

```
card-news-pipeline-project/
├── CLAUDE.md                  # 오케스트레이터 지시서
├── README.md                  # 프로젝트 개요
├── feedback.md                # 취향 저장소
├── agents/                    # AI 에이전트
│   ├── ct-researcher.md
│   ├── ct-cardnews-maker.md
│   └── ct-theme-designer.md
├── skills/                    # 실행 스킬 (20개+)
├── pipelines/                 # 시리즈별 실행 순서
│   ├── pipeline-A.md
│   ├── pipeline-B.md
│   └── pipeline-C.md
├── templates/                 # HTML 슬라이드 템플릿
├── scripts/                   # 자동화 스크립트
└── output/                    # 산출물 저장소
```

### 시작 명령 예시

```
"A시리즈 - AI 마케팅 자동화로 카드뉴스 만들어줘"
"B시리즈 - API 카드뉴스 해줘"
"C시리즈 - [원문 텍스트 붙여넣기]"
"D시리즈 - [주제]로 뉴스레터 써줘"
```

## 3. 주요 구성 요소

### 에이전트 (Agents)

| 에이전트 | 역할 | 출력 |
|---------|------|------|
| `ct-researcher` | 웹 검색으로 팩트, 트렌드, 사례 수집 | `output/research/research-[주제].md` |
| `ct-cardnews-maker` | HTML 슬라이드 디자인 및 제작 | 1080×1080px 슬라이드 |
| `ct-theme-designer` | 색상, 폰트, 레이아웃 테마 설계 | 시리즈별 테마 |

### 주요 스킬 (Skills)

| 스킬 | 역할 |
|------|------|
| `brief-writer` | 방향성 + 제목 3안 제시 |
| `column-writer` | 1500~2000자 칼럼 작성 |
| `text-formatter` | 슬라이드용 줄바꿈/강조 처리 |
| `caption-writer` | SNS 캡션 자동 작성 |
| `longform-writer` | 롱폼 콘텐츠 작성 |
| `thread-writer` | 스레드/X 포스트 작성 |
| `validate-slide` | QA 검증 (품질 체크) |
| `html-to-png` | HTML → PNG 이미지 변환 |

### 파이프라인 비교

| 항목 | A시리즈 | B시리즈 | C시리즈 | D시리즈 |
|------|--------|--------|--------|--------|
| 컨펌 횟수 | 3회 | 1회 | 1회 | 2회 |
| 소요 시간 | 30~60분 | 15~30분 | 10~20분 | 20~40분 |
| 리서치 | O | O | X | O |

## 4. 사용 방법

### A시리즈: 코딩하는 마케터

**입력**: `"A시리즈 - [주제/키워드]로 카드뉴스 만들어줘"`

**단계**:
1. 웹 리서치
2. 방향성 + 제목 3안 제시 → **컨펌 1**
3. 칼럼 작성
4. 산출물 선택 (카드뉴스/롱폼/스레드) → **컨펌 2**
5. 산출물 제작 + QA 검증
6. 최종 확인 → **컨펌 3**
7. PNG 추출 + 캡션 작성

**결과물**: 칼럼 기반 카드뉴스 8~10장 + 선택 산출물

---

### B시리즈: 알쓸IT잡

**입력**: `"B시리즈 - [IT 용어] 카드뉴스 해줘"`

**단계**:
1. IT 용어 리서치
2. 비주얼 슬라이드 제작
3. QA 검증
4. 최종 확인 → **컨펌**
5. PNG + MP4 추출

**결과물**: 비주얼 카드뉴스 5~7장 + MP4 표지

---

### C시리즈: 커리어 레터

**입력**: `"C시리즈 - [원문 텍스트 붙여넣기]"`

> ⚠️ 리서치 없음. 원문 내용 수정 금지.

**단계**:
1. 텍스트 포맷팅 (원문 보존)
2. 시네마틱 슬라이드 제작
3. 최종 확인 → **컨펌**
4. PNG 추출

**결과물**: 원문 보존 카드뉴스

---

### D시리즈: 코마레터

**입력**: `"D시리즈 - [주제]로 뉴스레터 써줘"`

**단계**:
1. 리서치
2. 칼럼 작성
3. 뉴스레터 본문 → **컨펌 1**
4. 멀티채널 배포본 제작 (인스타/링크드인/스레드/이메일)
5. 최종 확인 → **컨펌 2**

**결과물**: 뉴스레터 + 멀티채널 배포 콘텐츠

## 5. 출력 결과

### 산출물 저장 위치

```
output/
├── research/      # 리서치 파일
├── columns/       # 칼럼, 롱폼, 스레드
├── slides/        # HTML 슬라이드
├── images/        # PNG / MP4
└── captions/      # SNS 캡션
```

### 채널별 필요 파일

| 채널 | 필요 파일 |
|------|---------|
| 인스타그램 | `output/images/slide-*.png` + `output/captions/caption-*.txt` |
| 링크드인 | `output/columns/longform-*.md` |
| 스레드/X | `output/columns/thread-*.md` |
| 뉴스레터 | `output/columns/newsletter-*.md` |

## 6. 자주 묻는 질문 (FAQ)

**Q. 처음부터 다시 시작하려면?**
→ `output/` 폴더 파일 삭제 후 새 요청

**Q. 특정 단계만 다시 실행하고 싶어요**
→ 해당 스킬을 Claude Code에서 직접 호출

**Q. 슬라이드가 너무 길어요/짧아요**
→ `brief-writer` 컨펌 단계에서 수정 요청

**Q. 피드백이 반영이 잘 안 돼요**
→ `feedback.md` 파일 내용 업데이트 확인

**Q. 에러가 발생했어요**
→ 오케스트레이터에 에러 메시지와 함께 알리기

**Q. 이 시스템을 확장하고 싶어요**
→ `skills/` 폴더에 새 스킬 추가 후 `CLAUDE.md`에 호출 추가

## 7. 팁 & 베스트 프랙티스

### 좋은 결과를 위한 팁

- **주제 선정**: 구체적이고 최신 이슈 또는 실제 경험 기반
- **컨펌 피드백**: 명확하고 구체적으로, 여러 수정 사항 한 번에 전달
- **산출물 선택**: 한 번에 너무 많으면 시간이 오래 걸림 → 필요한 것만 선택

### 주간 워크플로우 예시

```
월요일 → D시리즈 (뉴스레터)
수요일 → A시리즈 (칼럼 기반 카드뉴스)
금요일 → B시리즈 (IT 용어 카드뉴스)
```

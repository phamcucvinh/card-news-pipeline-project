# 마케터 임지은의 카드뉴스 자동화 파이프라인

> **Claude Code + Multi-Agent System**
> AI 에이전트 팀이 리서치부터 카드뉴스 제작, PNG 추출까지 전 과정을 자동으로 처리합니다

출처: https://imployee.github.io/cardnews-pipeline/

---

## 시스템 구조

오케스트레이터 1명 + 전문 서브에이전트 팀

### 🎬 디렉터 (오케스트레이터)
전체 워크플로우를 관리하는 메인 에이전트. 직접 콘텐츠를 만들지 않고, 서브에이전트에 위임하며 JJ의 컨펌을 조율한다.

**역할**: 워크플로우 라우팅 · 컨펌 관리 · QA 검수

### 🤖 공식 서브에이전트 (3개)
전용 에이전트 파일에 역할·규칙·메모리가 내장된 전문가. 작업 지시만으로 독립 수행한다.

**에이전트**: `ct-researcher` · `ct-cardnews-maker` · `ct-theme-designer`

### ⚡ Executor 스킬 (9개)
역할별 지시서(SKILL.md)를 받아 실행하는 범용 에이전트. 스킬 파일이 곧 프롬프트.

**스킬**: `brief-writer` · `column-writer` · `text-formatter` · `caption-writer` · `longform` · `thread`

## 제작 파이프라인

### A. 코딩하는 마케터

> 칼럼 기반 — 가장 풍부한 파이프라인. 리서치 → 칼럼 → 카드뉴스/롱폼/스레드

| 단계 | 담당 | 산출물 |
|------|------|--------|
| 👤 주제/키워드 입력 | JJ 입력 | — |
| 1. 리서치 | 🤖 ct-researcher | 📄 research.md |
| 2. 방향성 + 제목 3안 제시 | ⚡ brief-writer | — |
| ⭐ **컨펌 1** — 방향성 + 제목 확정 | 👤 JJ 승인 | — |
| 3. 칼럼 작성 | ⚡ column-writer | 📄 column.md |
| ⭐ **컨펌 2** — 칼럼 확정 + 산출물 선택 | 👤 JJ 승인 + 선택 | — |
| 4. 산출물 제작 (병렬 가능) | 에이전트 분기 | HTML 슬라이드 / 롱폼 / 스레드 |
| 🔄 QA 검수 (자동) | 🔍 자동 실행 | — |
| ⭐ **컨펌 3** — 최종 결과물 검토 | 👤 JJ 승인 | — |
| 🔄 파일 추출 (자동) | ⚙️ html_to_png.sh + ⚡ caption-writer | 🖼️ slide-*.png / 📄 caption.txt |
| ✦ 마무리 | 💾 save-to-notion | 📄 feedback.md |

**산출물 제작 분기:**
- 📊 **카드뉴스** — text-formatter → ct-cardnews-maker → HTML 슬라이드
- 📝 **롱폼** — longform-writer → longform.md
- 🧵 **스레드** — thread-writer → thread.md

### B. 알쓸IT잡 (IT단어장)

| 단계 | 담당 | 산출물 |
|------|------|--------|
| 👤 IT 용어/키워드 입력 | JJ 입력 | — |
| 1. 리서치 | 🤖 ct-researcher | 📄 research.md |
| 2. 카드뉴스 제작 (바로 진행) | 🤖 ct-cardnews-maker | 📊 slide-*.html |
| 🔄 QA 검수 (자동) | 🔍 자동 실행 | — |
| ⭐ **컨펌** — 유일한 확인 포인트 | 👤 JJ 승인 | — |
| 🔄 파일 추출 (자동) | ⚙️ Playwright | 🖼️ PNG + 🎬 MP4 |
| ✦ 마무리 | — | 📄 feedback.md |

## 핵심 설계 원칙

### 🔗 Human-in-the-Loop
AI가 단독으로 완성하지 않는다. 핵심 분기점마다 JJ가 직접 방향을 결정하고 승인한다. 자동화와 통제의 균형.

### 🧩 역할 분리 + 조합
12개 스킬을 레고처럼 조합한다. 시리즈마다 필요한 스킬만 골라 파이프라인을 구성. 새 시리즈도 기존 스킬 재활용으로 즉시 런칭.

### 📚 누적 학습 (feedback.md)
매 제작마다 JJ 피드백을 수집하여 feedback.md에 누적. 모든 에이전트가 이를 참조해 점점 JJ 취향에 가까워진다.

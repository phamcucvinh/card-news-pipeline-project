# 디렉터 — 카드뉴스 파이프라인 오케스트레이터

## 역할
전체 카드뉴스 제작 워크플로우를 관리하는 메인 에이전트.
직접 콘텐츠를 만들지 않고, 서브에이전트에 위임하며 JJ의 컨펌을 조율한다.

## 핵심 원칙
1. **Human-in-the-Loop**: 핵심 분기점마다 반드시 JJ에게 확인을 받는다
2. **역할 위임**: 콘텐츠 제작은 항상 전문 에이전트/스킬에 위임한다
3. **feedback.md 참조**: 모든 작업 전 `feedback.md`를 읽어 JJ 취향을 반영한다
4. **산출물 추적**: 각 단계 완료 시 `output/` 폴더에 결과물을 저장한다
5. **QA 의무**: 최종 전달 전 반드시 validate_slide.py 실행

## 시리즈별 파이프라인 라우팅

JJ가 시리즈를 지정하면 해당 파이프라인 파일을 읽고 순서대로 실행:

| 시리즈 | 이름 | 파이프라인 파일 | 복잡도 |
|--------|------|----------------|--------|
| A | 코딩하는 마케터 | `pipelines/pipeline-A.md` | 높음 (컨펌 3회) |
| B | 알쓸IT잡 | `pipelines/pipeline-B.md` | 낮음 (컨펌 1회) |
| C | 커리어 레터 | `pipelines/pipeline-C.md` | 낮음 (컨펌 1회) |
| D | 코마레터 (뉴스레터) | `pipelines/pipeline-D.md` | 중간 (컨펌 2회) |

## 시작 방법

JJ가 다음과 같이 요청하면 즉시 파이프라인을 시작:

```
"A시리즈 - [주제]로 카드뉴스 만들어줘"
"B시리즈 - [IT용어] 카드뉴스 해줘"
"C시리즈 - [원문 텍스트 붙여넣기]"
"D시리즈 - [주제]로 뉴스레터 써줘"
```

## 컨펌 요청 형식

컨펌이 필요한 단계에서 반드시 아래 형식으로 JJ에게 제시:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🎬 [단계명] 완료 — JJ 확인 필요
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[내용 요약 또는 결과물 미리보기]

✅ 승인하면 → 다음 단계 바로 진행합니다
✏️ 수정하면 → 수정할 내용을 말씀해 주세요
```

## 에이전트/스킬 호출 방법

### 공식 서브에이전트 호출
```
Task("agents/ct-researcher.md의 지시에 따라 [주제]를 리서치해줘. 결과를 output/research/research-[주제]-YYYYMMDD.md에 저장해줘")
```

### Executor 스킬 호출
```
Task("skills/column-writer.md의 지시에 따라 칼럼을 작성해줘. 입력: output/research/research-XXX.md, 방향: [방향], 제목: [제목]. 결과를 output/columns/column-[주제]-YYYYMMDD.md에 저장해줘")
```

## feedback.md 업데이트 규칙

작업 완료 후 반드시:
1. JJ에게 회고 요청: "이번 카드뉴스 어땠나요? 다음에 개선할 점 있으면 알려주세요 😊"
2. 응답 받으면 `feedback.md`에 날짜 + 시리즈 + 피드백 내용 추가
3. 다음 작업 시 자동 반영

## 파일 구조 참조

```
card-news-pipeline-project/
├── CLAUDE.md          ← 지금 이 파일 (오케스트레이터)
├── feedback.md        ← JJ 취향 누적 저장소
├── agents/            ← 서브에이전트 정의 (4개)
├── skills/            ← Executor 스킬 정의 (14개)
├── pipelines/         ← 시리즈별 실행 순서
├── templates/         ← HTML 슬라이드 템플릿
├── scripts/           ← 자동화 스크립트
└── output/            ← 모든 산출물 저장
    ├── research/
    ├── columns/
    ├── slides/
    ├── images/
    └── captions/
```
